Detailed Description

Struct to be filled by the user with data describing the sink stage.

This struct is to be filled by the user with data describing how the sink will process the buffers, together with some additional data the sink may use. Not all of the fields are to be provided: check the documentation of the fields for additional information.

Member Data Documentation

◆ init

mtdp_sink_callback mtdp_sink::init

Sink initialization function.

This function, if set, will be called only once after the pipeline will be enabled right before the first sink iteration. It may be used e.g. for initializing scratch data directly from within the sink callback on/with the self parameter. It is optional to set.

◆ name

const char* mtdp_sink::name

This field will name the thread running the sink callback.

Naming a thread is useful for both debugging and production runtime monitoring. It is optional to set: by default, the thread name will not be set, and the default behavior is platform dependent (often it will inherit the name of the thread enabling the pipeline).

◆ process

mtdp_sink_callback mtdp_sink::process

Callback to be called by the sink stage on every iteration.

This callback will be called on a context containing freshly produced buffers - when the previous stages will provide any-, together with some user data. It is required to set, since it is the core of the whole processing mechanism.

Note
Remember to provide a function that repeatedly sets the ready_to_pull parameter in the sink context or the input buffers will not be updated and the sink thoughput will be zeroed.

◆ self

mtdp_sink_data mtdp_sink::self

User data provided to the sink.

Even though for simple stages this may be an overkill, bigger and more complex stages may incorporate mechanisms to communicate partial results to external threads, or even just to provide a configuration mechanism. It is optional to set.
