Detailed Description

Struct to be filled by the user with data describing an internal stage.

This struct is to be filled by the user with data describing how the stage will process the buffers, together with some additional data the stage may use. Not all of the fields are to be provided: check the documentation of the fields for additional information.

Member Data Documentation

◆ init

mtdp_stage_callback mtdp_stage::init

Stage initialization function.

This function, if set, will be called only once after the pipeline will be enabled right before the first stage iteration. It may be used e.g. for initializing scratch data directly from within the stage callback on/with the self parameter. It is optional to set.

◆ name

const char* mtdp_stage::name

This field will name the thread running the stage callback.

Naming a thread is useful for both debugging and production runtime monitoring. It is optional to set: by default, the thread name will not be set, and the default behavior is platform dependent (often it will inherit the name of the thread enabling the pipeline).

◆ process

mtdp_stage_callback mtdp_stage::process

Callback to be called by the stage on every iteration.

This callback will be called on a context containing freshly produced input buffers - when the previous stages will provide any -, with an empty output buffer obtained by the output pipe, and with user-provided data. It is required to set, since it is the core of the whole processing mechanism.

Note
Remember to provide a function that repeatedly sets the ready_to_pull parameter in the stage context or the input buffers will not be updated and the stage thoughput will be zeroed.

◆ self

mtdp_stage_data mtdp_stage::self

User data provided to the stage.

Even though for simple stages this may be an overkill, bigger and more complex stages may incorporate mechanisms to communicate partial results to external threads, or even just to provide a configuration mechanism. It is optional to set.
